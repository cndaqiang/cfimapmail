# 使用手册

仓库地址：https://github.com/cndaqiang/cfimapmail

## 基本使用

1. 打开首页
2. 输入 IMAP 地址，例如 `imap.example.com`
3. 输入端口，推荐 `993`
4. 输入邮箱和密码
5. 选择要拉取的最近邮件数量，默认 5，最大 50
6. 点击“拉取最近邮件”

页面会先显示邮件列表。列表包含：

- 标题
- 发件人
- 收件人
- 抄送
- 回复地址
- 时间
- Message-ID
- plain text 开头预览

列表阶段只读取每封邮件 plain text 正文开头的一小段，不会一次拉取完整正文。

## 查看正文

点击某一封邮件卡片后，页面会再次请求 Worker。

Worker 会按该邮件的 UID 定向读取 `text/plain` 正文，并展开显示在页面中。正文使用 plain text 显示，尽量保留原本换行。

如果邮件只有 HTML，没有 `text/plain`，页面会显示“未找到 plain text 正文”。当前阶段不渲染 HTML。

## IMAP 参数

- **IMAP 地址**：邮箱服务商提供的 IMAP 服务器地址
- **端口**：第一版推荐 `993`，即 IMAPS/TLS 直连
- **邮箱**：完整邮箱地址
- **密码**：推荐使用应用专用密码，不建议使用主账号密码
- **数量**：拉取最近多少封邮件的元信息和预览，Worker 会限制最大值，避免请求过慢

## 本地保存

- 勾选“记住本机账号和密码”后，页面会把 IMAP 地址、端口、邮箱和密码保存到当前浏览器
- 下次打开页面会自动填充
- 点击“清空本地保存”可以删除这些信息

> 本地保存使用 localStorage。密码会存放在当前浏览器中，只建议在自己的可信设备上启用。

## 当前读取范围

当前阶段只读取：

- 邮件展示元信息：From、To、Cc、Reply-To、Sender、Subject、Date、Message-ID
- MIME 结构：用于定位 `text/plain`
- plain text 预览
- 点击后读取 plain text 正文

当前阶段不读取：

- 附件
- 图片
- HTML 正文
- 整封原始邮件

## 常见问题

### 登录失败

请检查：

- 邮箱是否启用了 IMAP
- 是否需要应用专用密码
- IMAP 地址和端口是否正确
- 是否使用了 993 端口

### 没有正文预览或正文

部分邮件可能只有 HTML，没有 `text/plain`；也可能 MIME 结构比较特殊。当前版本只定位并读取 `text/plain`，不会回退去拉 HTML。

### 正文没有完全显示

点击邮件后，Worker 默认只读取 plain text 正文前 64KB，避免大邮件导致请求过慢。后续可以扩展“继续加载更多”。

### 端口 143 能用吗

第一版按 IMAPS/TLS 直连实现，推荐使用 993。STARTTLS 或明文 143 不作为默认支持目标。
